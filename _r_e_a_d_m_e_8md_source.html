6 A Modelica compiler written in modern C++ with Python bindings.
7 
8 ## Features
9 
10 * Parsing using [ANTLR4 C++ Runtime](https://github.com/antlr/antlr4/blob/master/doc/cpp-target.md).
11  * <img src="doc/ast.png" alt="drawing" width="500px"/>
12 * Listener based generators decoupled from the AST, easy to compile Modelica to new targets.
13  * Lisp string printing [LispListener](src/cymoca/ast/listener/lisp.h).
14 * Listener based compiler
15  * [Compiler](src/cymoca/compiler.cc)
16 * C++17 used throughout, minimal copying, low memory footprint, and fast.
17 * Python bindings planned.
18 * Data format for simulations [HDF5](https://www.hdfgroup.org/)
19

63 ### Build from Source
64 
65 * You can build cymoca from source using cmake. You will need the following dependencies:
66 
67 * Dependencies
68  * Antlr4 C++ runtime:
69  * Ubuntu 18.04: [deb](https://github.com/jgoppert/antlr4/releases/download/4.7.1-cpack/LIBANTLR4-4.7.1-Linux.deb)
70  * Mac [Zip](http://www.antlr.org/download/antlr4-cpp-runtime-4.7.1-macos.zip)
71  * Windows [Zip](http://www.antlr.org/download/antlr4-cpp-runtime-4.7.1-vs2015.zip)
72  * Build from Source: see [script](scripts/install_antlr.sh)
73  * Casadi, optional, requires system installed headers and libraries, enabled with -DWITH_CASADI=ON
74  * Ubuntu 18.04: [deb](https://github.com/jgoppert/casadi/releases/download/3.4.4-jmg/casadi-3.4.4-Linux.deb)
75  * Build from Source: see [script](scripts/install_casadi.sh)
76  * Gtest, optional, used for testing
77  * Build from Source: see [script](scripts/install_googletest.sh)
78 
79 * A quick example of building with cmake:
80 ```bash
81 git clone https://github.com/jgoppert/cymoca
82 cd cymoca
83 mkdir -p build
84 cd build && cmake -DWITH_TEST=ON ..
85 make -j4
86 ctest
87 cpack -G DEB
88 sudo dpkg -i cymoca*.deb
89 cymoca my_model.mo
90 ```
